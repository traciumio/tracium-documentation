Tracium Engine

The Code State Engine (CSE). Executes code, captures every runtime state change through JDI, normalizes it into a truth-first timeline, and emits UEF.

Execution Pipeline

Source Input

Java code snippet or file

→

Compile

javac compilation

→

Execute + JDI

Run with debugger attached

→

Capture

Raw runtime events

→

Normalize

State engine processing

→

Timeline

Ordered steps + deltas

→

UEF Output

Serialized trace

Engine Modules

engine-core

The intellectual center of the engine.

  • State Engine (stack, heap, references)
  • Timeline Engine (ordered steps, checkpoints)
  • Event classification and normalization
  • UEF serialization
  • Execution adapter interface

engine-java-jdi-adapter

Java-specific execution through JDI.

  • JVM launch and JDI connection
  • Method entry/exit event capture
  • Line-level step events
  • Variable and field inspection
  • Object identity tracking
  • Sandbox enforcement

engine-service

Deployable service layer.

  • Session management (create/query/destroy)
  • Async execution handling
  • Status tracking (PENDING/RUNNING/COMPLETE)
  • Result retrieval API
  • Health check endpoint
