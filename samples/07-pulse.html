Host Rollout

Current Host

VS Code

Pulse starts in VS Code because it gives the fastest extension loop and a strong webview-based surface for trace playback and architecture exploration.

  • TypeScript + VS Code Extension API
  • Trace sidebar, inline values, and architecture webviews
  • Fast feedback loop for shaping the Pulse workflow
Planned Host

JetBrains

JetBrains is the next Pulse host, especially important for IntelliJ IDEA and Java-heavy teams using Tracium Engine and Atlas.

  • Kotlin + IntelliJ Platform SDK
  • Tool windows for trace and architecture workflows
  • Project-aware navigation for Java-first Tracium use cases

Key Features

Inline Value Decorations

See variable values right next to the code as you step through execution. Values update in real-time as you navigate the trace.

Trace Sidebar

Full trace step list, call stack, and variable inspector in the VS Code sidebar. Click any step to jump to that point in execution.

Multi-Host Product

Pulse is one IDE product across multiple hosts. VS Code is first, and JetBrains is planned next with the same Tracium workflows adapted to each IDE.

JetBrains Tool Window Path

JetBrains support will use tool windows and project-aware actions so Java tracing and repository analysis feel native inside IntelliJ IDEA.
